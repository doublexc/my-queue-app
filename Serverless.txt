รายงานสรุปโครงสร้างและประเด็นเทคนิคเว็บแอปจัดการคิว (Serverless)
ความสามารถของเว็บแอป:

• โครงสร้างระบบ: ใช้ GitHub Pages เป็นหน้าเว็บ (Frontend) และใช้ Google Sheets เป็นฐานข้อมูลผ่าน Google Apps Script (Backend) แบบไม่มีค่าใช้จ่ายเซิร์ฟเวอร์

• ระบบสร้างคิว (ฝั่งผู้จัดการ):

  • สร้างรหัสคิว 6 หลักอัตโนมัติ

  • ควบคุมลำดับคิวปัจจุบัน (เพิ่ม/ลด) และแสดงผลแบบ Real-time ให้ผู้ใช้อื่นเห็น

  • มีระบบยกเลิกคิว (ล้างข้อมูลแถวทั้งหมด) เพื่อจบการทำงาน

• ระบบต่อคิว (ฝั่งผู้ใช้):

  • แสดงลำดับปัจจุบันและคำนวณเวลาเฉลี่ยต่อคิว (คิดจากคนแรกถึงคนล่าสุด)

  • แสดงเวลาคาดการณ์และเวลาที่เหลือที่จะถึงลำดับของตนเอง

  • ระบบความปลอดภัย: รหัสยืนยัน 24 หลัก (Secret Key) พร้อมปุ่ม Copy เพื่อป้องกันการแอบอ้างและการหลุดออกจากหน้าเว็บ

ปัญหาและข้อจำกัดที่พบ/ต้องระวัง:

• Race Condition (การกดคิวซ้อน): หากมีการกดต่อคิวพร้อมกันในระดับวินาที อาจเกิดปัญหาลำดับซ้ำกัน แก้ไขได้โดยใช้ `LockService` ใน Google Apps Script

• ข้อจำกัดของ Google Quotas:

  • การทำงานพร้อมกัน (Simultaneous Executions) จำกัดที่ 30 ครั้ง

  • การเรียกใช้ URL Fetch (API) จำกัด 20,000 ครั้งต่อวัน

• การแจ้งเตือน: การใช้ LINE Notify หรือ LINE Group มีความยุ่งยาก (High Friction) สำหรับผู้ใช้ทั่วไป จึงเน้นการใช้ Browser Polling (ดึงข้อมูลอัตโนมัติทุกช่วงเวลา) และ Web Speech API (ส่งเสียงเตือน) แทน
